# PRODUCT.md

## 제품 요약
YouthFit은 흩어진 청년 정책 정보를 한곳에 모으고, 사용자가 자신의 상황에 정책이 맞을 가능성이 있는지 이해하도록 돕는 서비스다.

핵심 사용자 가치:
1. 관련 정책을 빠르게 찾을 수 있다.
2. 어려운 정책 문구를 쉬운 말로 이해할 수 있다.
3. 원문 공고를 전부 읽지 않아도 다음 행동을 알 수 있다.

## 문제 정의
사용자가 어려움을 겪는 이유:
- 청년 정책이 여러 공식·비공식 채널에 흩어져 있다.
- 행정 용어를 개인의 실제 상황에 대입하기 어렵다.
- 정책의 존재를 알아도 신청 경로가 불명확해 포기하는 경우가 많다.

## 타겟 사용자
주요 타겟:
- 만 24세~29세
- 사회 초년생, 취업 준비생, 최근 졸업자
- 소득, 고용, 주거, 가구 상황이 자주 변하는 사용자

## 제품 포지셔닝
YouthFit은 공식 신청 시스템을 대체하지 않는다.

역할은 **탐색 및 해석 레이어**다:
- 더 쉬운 탐색
- 더 쉬운 이해
- 가벼운 적합도 가이드
- 출처 기반 Q&A
- 최종 신청은 공식 채널로 연결

## MVP 기능
### v0 포함 범위
- 필터 및 정렬이 가능한 정책 목록
- 정책 상세 보기
- 키워드 및 의미 기반 검색
- 카카오 로그인
- 사용자 프로필 생성 및 수정
- 적합도 판정
- 인덱싱된 원문 기반 정책 Q&A
- 북마크
- 마감일 이메일 알림

### 후속 버전으로 이관
- 커뮤니티 기능
- 평점 및 댓글
- 모바일 푸시 알림
- 관리자 대시보드
- 고급 하이브리드 검색
- 수집 및 후속 처리용 이벤트 드리븐 아키텍처

## 적합도 판정 원칙
v0의 적합도 판정은 **법적 효력이 있는 자격 판정**이 아니라, 가벼운 해석 보조 기능으로 취급한다.

출력은 아래 표현을 우선 사용한다:
- `해당 가능성 높음`
- `불명확 / 추가 확인 필요`
- `해당 가능성 낮음`

각 결과에는 반드시 포함한다:
- 주요 판단 이유
- 누락되었거나 불명확한 조건
- 판단 근거가 된 원문 필드 또는 공고 섹션

## Q&A 원칙
Q&A는 반드시 인덱싱된 원문에 근거해야 한다.

시스템은 다음을 지켜야 한다:
- 출처 기반으로 답변한다.
- 원문에 없는 요구사항을 지어내지 않는다.
- 원문이 불완전하거나 모호하면 불확실성을 인정한다.
- 과도한 해석보다 관련 공고 구간의 인용·근거 제시를 우선한다.

## 사용자 신뢰 원칙
- 쉽게 설명하되, 중요한 조건을 지나치게 단순화하지 않는다.
- 판정이 불확실하면 명확하게 알린다.
- 최종 제출 전에는 공식 신청 채널에서 다시 확인하도록 유도한다.
