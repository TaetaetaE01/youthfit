# LoginPage — 로그인 페이지 디자인 스펙

> 라우트: `/login`
> 접근 권한: 비로그인 사용자 전용 (로그인 시 리다이렉트)
> 관련 기능: F-04 (카카오 소셜 로그인)

---

## 1. 페이지 목적

카카오 소셜 로그인 단일 방식으로 인증 마찰을 최소화하고, 로그인 후 원래 페이지로 복귀시킨다.
PRD 전환 지표 "비로그인 → 로그인 전환율" 직결.

---

## 2. 사용자 플로우

```
로그인 필요 기능 클릭 (적합도/Q&A/북마크)
  → LoginPage 이동 (?redirect=/policies/123 파라미터 보존)
  → "카카오로 시작하기" 클릭
  → 카카오 OAuth 인증 페이지 (외부)
  → 콜백 → JWT 발급 → redirect URL로 이동
```

### 콜백 처리 (별도 라우트)
```
라우트: /auth/kakao/callback
  → URL에서 인가 코드(code) 추출
  → POST /api/auth/kakao { "code": "..." }
  → JWT 수신 → Zustand 스토어에 저장
  → redirect URL로 이동 (없으면 /policies)
```

## 3. 카피

### 로그인 카드
- 로고: **YouthFit** (아이콘 + 텍스트)
- 타이틀: **로그인**
- 설명: **카카오 계정으로 간편하게 시작하세요**
- 카카오 버튼: **카카오로 시작하기** (카카오 로고 포함)

### 비로그인 탐색 유도
- 텍스트: **회원가입 없이도 정책을 둘러볼 수 있어요.**
- 링크: **정책 둘러보기** → `/policies`

### 개인정보 동의
- 텍스트: **로그인 시 [이용약관] 및 [개인정보처리방침]에 동의합니다.**

### 에러 상태
- 인증 실패: **로그인에 실패했어요. 다시 시도해주세요.**
- 네트워크 에러: **네트워크 연결을 확인해주세요.**

### 콜백 로딩
- 텍스트: **로그인 중이에요...**
- 스피너 표시 (이 페이지에서만 스피너 허용)

## 6. 인터랙션 & 상태

### 로그인 버튼 클릭
1. 카카오 OAuth 인증 URL로 리다이렉트
2. `redirect` searchParam이 있으면 state에 포함
3. 카카오 인증 완료 → 콜백 URL로 리다이렉트

### 콜백 처리
1. URL에서 `code` 파라미터 추출
2. `POST /api/auth/kakao` 호출
3. 성공: JWT 저장 → redirect URL 이동 (없으면 `/policies`)
4. 실패: LoginPage로 이동 + 에러 메시지 표시

### 이미 로그인 상태
- LoginPage 접근 시 즉시 `/policies`로 리다이렉트

### 에러 복구
- 에러 메시지 3초 후 자동 fade-out
- 버튼 재클릭 가능

## 8. 접근성

- 카카오 버튼: `aria-label="카카오 계정으로 로그인"`
- 에러 메시지: `role="alert"`, `aria-live="assertive"`
- 키보드: Tab으로 버튼·링크 순서대로 접근, Enter로 활성화
- 포커스: 카카오 버튼에 `secondary-500` 포커스 링 (`0 0 0 3px rgba(99,102,241,0.15)`)

---

## 9. API 연동

### 카카오 로그인
```
POST /api/auth/kakao
Content-Type: application/json
Body: { "code": "카카오_인가_코드" }
Response: { "success": true, "data": { "accessToken": "...", "refreshToken": "..." } }
```

### 토큰 저장
- accessToken: Zustand 스토어 (메모리) + localStorage
- refreshToken: httpOnly 쿠키 (가능한 경우) 또는 안전한 저장소
