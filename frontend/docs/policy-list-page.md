# PolicyListPage — 정책 목록 페이지 디자인 스펙

> 라우트: `/policies`
> 접근 권한: 비로그인 포함 전체
> 관련 기능: F-01 (정책 목록), F-03 (키워드 검색), F-08 (북마크)

---

## 1. 페이지 목적

정책을 필터·검색·정렬하여 탐색하고, 관심 정책의 상세 페이지로 진입하는 허브 페이지.
PRD 전환 지표 "검색 결과 클릭률(CTR)"을 직접 좌우한다.

---

## 2. 사용자 플로우

```
진입 (GNB "정책 목록" 또는 랜딩 CTA "정책 둘러보기")
  → 필터/검색으로 범위 축소
  → 정책 카드 스캔
  → 카드 클릭 → PolicyDetailPage
```

---

## 3. 카피

### 페이지 헤더
- 타이틀: **청년 정책 찾기**
- 서브카피: 나에게 맞는 청년 정책을 찾아보세요.

### 검색바
- placeholder: **"키워드로 정책을 검색하세요"**
- 버튼: **검색**

### 필터 라벨
- 카테고리: **카테고리**
- 지역: **지역**
- 모집상태: **모집 상태**
- 전체 해제: **필터 초기화**
- 모바일 필터 버튼: **필터**

### 카테고리 옵션 (7개)
| 값 | 라벨 |
|----|------|
| JOBS | 취업 |
| HOUSING | 주거 |
| EDUCATION | 교육 |
| WELFARE | 복지 |
| FINANCE | 금융 |
| CULTURE | 문화 |
| PARTICIPATION | 참여 |

### 모집 상태 옵션 (3개)
| 값 | 라벨 | 배지 색상 |
|----|------|----------|
| UPCOMING | 예정 | `neutral-500` bg |
| OPEN | 모집중 | `success-100` bg + `success-500` text |
| CLOSED | 마감 | `neutral-300` bg + `neutral-500` text |

### 정렬 옵션
- **최신순** (기본, `sortBy=createdAt`, `ascending=false`)
- **마감임박순** (`sortBy=applyEnd`, `ascending=true`)

### 결과 메타
- 형식: **{totalElements}개 정책**
- 예시: "142개 정책"

### 빈 결과 상태
- 타이틀: **검색 결과가 없습니다**
- 설명: 다른 키워드나 필터로 다시 시도해보세요.
- 버튼: **필터 초기화**

### 로딩 상태
- 카드 스켈레톤 6개 표시 (스피너 사용 금지)


- **카테고리 배지**: `primary-100` bg, `secondary-600` text, `label-sm`, pill

- **요약**: `body-md` (14px), `neutral-700`, `line-clamp-2`
- **메타**: `caption` (12px), `neutral-500`, 지역 + 신청기간

## 6. 인터랙션 & 상태

### 검색
- 입력 시 300ms 디바운스 후 검색 실행
- Enter 키 및 "검색" 버튼 클릭 모두 지원
- 검색 시 `page=0`으로 리셋

### 필터
- 필터 변경 시 즉시 API 재호출 (`page=0` 리셋)
- 선택된 필터는 URL searchParams에 반영 (뒤로가기 시 복원)
- 모바일: "필터" 버튼 → 하단 시트(Bottom Sheet) 열림 → 적용/초기화 버튼

### 정렬
- 드롭다운 변경 시 즉시 API 재호출 (`page=0` 리셋)

### 북마크
- 로그인 사용자: 하트 아이콘 토글 (즉시 API 호출)
- 비로그인: 클릭 시 토스트 "로그인 후 북마크할 수 있어요" + "로그인" 링크

### 로딩
- 초기 로딩: 카드 스켈레톤 6개
- 필터/검색 변경 시: 기존 카드 fade-out → 스켈레톤 → 새 카드 fade-in

### 에러
- API 에러 시: "정책을 불러오지 못했습니다" + "다시 시도" 버튼

## 8. 접근성

- 필터 체크박스: `aria-label="카테고리: 취업"` 형식
- 카드: `<article>` 태그, 제목은 `<h3>`
- 페이지네이션: `aria-label="페이지 이동"`, 현재 페이지 `aria-current="page"`
- 검색바: `role="search"`, 입력 필드 `aria-label="정책 키워드 검색"`
- 빈 결과: `role="status"`, `aria-live="polite"`
- 키보드: 모든 인터랙티브 요소 Tab 접근 가능, Enter/Space로 활성화

## 9. API 연동

### 정책 목록 조회
```
GET /api/v1/policies?category={}&regionCode={}&status={}&sortBy={}&ascending={}&page={}&size=20
```

### 키워드 검색
```
GET /api/v1/policies/search?keyword={}&page={}&size=20
```

### 북마크 추가/삭제
```
POST /api/v1/bookmarks { "policyId": 1 }
DELETE /api/v1/bookmarks/{bookmarkId}
```

## 10. URL 상태 관리

모든 필터/검색/정렬/페이지 상태는 URL searchParams로 관리한다:
```
/policies?keyword=장학금&category=EDUCATION&regionCode=SEOUL&status=OPEN&sortBy=applyEnd&ascending=true&page=1
```

이를 통해:
- 뒤로가기/앞으로가기 시 상태 복원
- URL 공유 시 동일한 검색 결과 재현
- 새로고침 시 상태 유지
