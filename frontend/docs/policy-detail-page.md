# PolicyDetailPage — 정책 상세 페이지 디자인 스펙

> 라우트: `/policies/:policyId`
> 접근 권한: 비로그인 포함 전체 (적합도·Q&A는 로그인 필요)
> 관련 기능: F-02 (정책 상세), F-06 (적합도 판정), F-07 (정책 Q&A), F-08 (북마크)

---

## 1. 페이지 목적

단일 정책의 전체 정보를 제공하고, AI 요약·적합도 판정·Q&A로 이해를 돕고, 최종 행동(공식 채널 신청)으로 연결하는 전환 페이지.
PRD 전환 지표 "상세 → 공식 신청 채널 이동률"의 핵심.

---

## 2. 사용자 플로우

```
PolicyListPage 카드 클릭 → 상세 진입
  → AI 요약으로 빠른 이해
  → 적합도 확인 (로그인 필요)
  → Q&A로 추가 질문 (로그인 필요)
  → 공식 신청 채널 버튼 클릭 → 외부 이동
```

## 3. 카피

### 브레드크럼
- 형식: **정책 목록** > **{정책 제목}**

### AI 요약 섹션
- 태그배지: **AI SUMMARY**
- disclaimer: **AI가 정리한 요약이에요. 정확한 내용은 공식 공고에서 확인해주세요.**

### 적합도 판정 섹션
- 섹션 타이틀: **내 적합도 확인**
- 버튼 (미판정): **내 적합도 확인하기**
- 버튼 (로딩): **판정 중...**
- 결과 등급:
  | 값 | 라벨 | 아이콘 | 색상 |
  |----|------|-------|------|
  | LIKELY_ELIGIBLE | 해당 가능성 높음 | ✓ 체크 | `success-500` |
  | UNCERTAIN | 추가 확인 필요 | ? 물음표 | `warning-500` |
  | LIKELY_INELIGIBLE | 해당 가능성 낮음 | ✕ 엑스 | `error-500` |
- disclaimer: **본 결과는 참고용이며, 법적 효력이 있는 자격 판정이 아닙니다. 최종 확인은 공식 신청 채널에서 진행해주세요.**
- 누락 필드 안내: **프로필에 {필드명}을 추가하면 더 정확한 판정이 가능해요.** + "프로필 수정" 링크
- 비로그인 안내: **로그인하면 적합도를 확인할 수 있어요.** + "카카오로 시작하기" 버튼

### 정책 상세 정보
- 탭(데스크톱) / 아코디언(모바일):
  - **신청 자격**
  - **신청 방법**
  - **제출 서류**
  - **문의처**

### Q&A 섹션
- 태그배지: **SMART Q&A**
- 입력 placeholder: **"이 정책에 대해 궁금한 점을 물어보세요"**
- 버튼: **질문하기**
- 로딩 단계:
  1. **원문을 검색하고 있어요...**
  2. **답변을 작성하고 있어요...**
- 출처 라벨: **출처**
- 비로그인 안내: **로그인 후 질문할 수 있어요.** + "카카오로 시작하기" 버튼

### 공식 채널 CTA
- 버튼: **공식 채널에서 신청하기** (외부 링크 아이콘 포함)
- 안내: **최종 신청은 공식 채널에서 확인해주세요.**

### 북마크
- 토스트 (추가): **북마크에 추가했어요**
- 토스트 (해제): **북마크를 해제했어요**
- 비로그인 토스트: **로그인 후 북마크할 수 있어요**

## 6. 인터랙션 & 상태

### 적합도 판정
1. **비로그인**: "로그인하면 적합도를 확인할 수 있어요" + "카카오로 시작하기" 인라인 CTA
2. **로그인 + 미판정**: "내 적합도 확인하기" Primary 버튼
3. **로딩**: 원형 프로그레스 스켈레톤 + "판정 중..." 텍스트
4. **판정 완료**: 원형 프로그레스 + 항목별 결과 리스트 + disclaimer
5. **누락 필드**: "프로필에 {필드}를 추가하면..." 안내 + 프로필 수정 링크

### Q&A
1. **비로그인**: 입력 비활성화 + "로그인 후 질문할 수 있어요" 안내
2. **입력 중**: 텍스트 입력 + "질문하기" 버튼 활성화
3. **로딩**: "원문을 검색하고 있어요..." → "답변을 작성하고 있어요..." 단계별 표시
4. **스트리밍**: SSE 기반 실시간 타이핑 표시
5. **완료**: 답변 + 출처 블록 표시
6. **에러**: "답변을 생성하지 못했어요. 다시 시도해주세요." + 재시도 버튼

### 북마크
- 로그인 사용자: 하트 아이콘 토글 (채워짐/비움) + 토스트
- 비로그인: 클릭 시 로그인 유도 토스트

## 8. 접근성

- 적합도 결과: 색상 외 아이콘+텍스트 병기 (색각 이상 사용자 배려)
- Q&A 채팅: `role="log"` + `aria-live="polite"` (스트리밍 답변 자동 읽기)
- 탭: `role="tablist"` + `role="tab"` + `role="tabpanel"`, 화살표 키 이동
- 아코디언: `aria-expanded`, `aria-controls`
- 외부 링크: `aria-label="공식 채널에서 신청하기 (새 탭에서 열림)"`
- 북마크: `aria-label="북마크 추가/해제"`, `aria-pressed`

## 9. API 연동

### 정책 상세 조회
```
GET /api/v1/policies/{policyId}
```

### 적합도 판정
```
POST /api/v1/eligibility/judge
Authorization: Bearer {accessToken}
Body: { "policyId": {policyId} }
```

### Q&A 질문 (SSE 스트리밍)
```
POST /api/v1/qna/ask
Authorization: Bearer {accessToken}
Accept: text/event-stream
Body: { "policyId": {policyId}, "question": "..." }
```

### 북마크
```
POST /api/v1/bookmarks { "policyId": {policyId} }
DELETE /api/v1/bookmarks/{bookmarkId}
```
