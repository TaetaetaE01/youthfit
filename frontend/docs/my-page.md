# MyPage — 마이페이지 디자인 스펙

> 라우트: `/mypage`
> 접근 권한: 로그인 필수 (비로그인 시 `/login?redirect=/mypage`로 이동)
> 관련 기능: F-05 (프로필), F-08 (북마크), F-09 (알림 설정)

---

## 1. 페이지 목적

프로필 관리, 북마크 모아보기, 알림 설정을 한 곳에서 제공하는 재방문 사용자 허브.
적합도 판정에 필요한 프로필 완성도를 높이는 유도 역할도 한다.

---

## 2. 사용자 플로우

```
GNB 프로필 아이콘/닉네임 클릭 → MyPage
  → 프로필 조회/수정
  → 북마크 목록 확인 → 정책 상세 이동
  → 알림 설정 변경
  → 로그아웃
```

---

## 3. 카피

### 프로필 헤더
- 닉네임: **{사용자 닉네임}** (예: "민지")
- 이메일: **{이메일}** (예: "minji@kakao.com")
- 수정 버튼: **프로필 수정**
- 저장 버튼: **저장**
- 취소 버튼: **취소**

### 프로필 완성도
- 프로그레스 라벨: **프로필 완성도 {n}%**
- 유도 문구: **프로필을 완성하면 적합도 판정이 더 정확해져요.**
- 버튼: **추가 정보 입력하기**

### 추가 정보 필드 (적합도 판정용)
| 필드 | 라벨 | 타입 | 예시 |
|------|------|------|------|
| age | 나이 | 숫자 입력 | 26 |
| region | 거주 지역 | 드롭다운 | 서울, 경기, ... (17개) |
| income | 월 소득 (만원) | 숫자 입력 | 80 |
| employmentStatus | 고용 상태 | 드롭다운 | 구직중, 재직중, 재학중, 기타 |
| isHouseholder | 세대주 여부 | 토글 | 예/아니오 |
| householdSize | 가구원 수 | 숫자 입력 | 1 |

### 탭
- 탭 1: **북마크** (기본 활성)
- 탭 2: **알림 설정**

### 북마크 탭
- 빈 상태 타이틀: **아직 북마크한 정책이 없어요**
- 빈 상태 설명: 관심 있는 정책을 북마크하면 여기에서 모아볼 수 있어요.
- 빈 상태 CTA: **정책 둘러보기** → `/policies`
- 마감 임박 배지: **D-{n}** (7일 이내)
- 해제 토스트: **북마크를 해제했어요** + **되돌리기** (3초)

### 알림 설정 탭
- 섹션 타이틀: **이메일 알림**
- 설명: **북마크한 정책의 마감일 전에 이메일로 알려드려요.**
- 토글 라벨: **이메일 알림 받기**
- 시점 라벨: **알림 시점**
- 시점 옵션:
  - **마감 3일 전**
  - **마감 7일 전** (기본)
  - **마감 14일 전**
- 저장 토스트: **알림 설정이 저장되었어요**

### 로그아웃
- 링크: **로그아웃**
- 확인 다이얼로그:
  - 타이틀: **로그아웃**
  - 설명: **로그아웃할까요?**
  - 확인 버튼: **로그아웃** (Destructive 스타일)
  - 취소 버튼: **취소**

## 6. 인터랙션 & 상태

### 프로필 수정
1. "프로필 수정" 클릭 → 편집 모드 (닉네임 입력 활성화)
2. 입력 → "저장" 클릭 → `PATCH /api/v1/users/me`
3. 성공: 편집 모드 해제 + 토스트 "프로필이 수정되었어요"
4. 실패: 에러 메시지 인라인 표시

### 추가 정보 입력
1. "추가 정보 입력하기" 클릭 → 폼 확장/모달 표시
2. 필드 입력 → "저장" → API 호출
3. 성공: 프로필 완성도 게이지 업데이트 + 토스트
4. 100% 완성 시: "프로필이 완성되었어요!" 축하 메시지

### 북마크 해제
1. 하트 아이콘 클릭 → `DELETE /api/v1/bookmarks/{id}`
2. 카드 fade-out + 토스트 "북마크를 해제했어요" + "되돌리기" 액션 (3초)
3. "되돌리기" 클릭 → `POST /api/v1/bookmarks` → 카드 복원

### 알림 설정 변경
1. 토글/라디오 변경 → 500ms 디바운스 후 `PUT /api/v1/notifications/settings`
2. 성공: 토스트 "알림 설정이 저장되었어요"
3. 실패: 이전 상태로 롤백 + 에러 토스트

### 로그아웃
1. "로그아웃" 클릭 → 확인 다이얼로그
2. "로그아웃" 확인 → `POST /api/auth/logout` → JWT 삭제 → `/` 이동

## 8. 접근성

- 프로필 이미지: `alt="{닉네임}의 프로필 이미지"`
- 토글 스위치: `role="switch"`, `aria-checked`, `aria-label="이메일 알림 받기"`
- 라디오 그룹: `role="radiogroup"`, 각 항목 `role="radio"`, `aria-checked`
- 프로필 완성도 바: `role="progressbar"`, `aria-valuenow`, `aria-valuemin=0`, `aria-valuemax=100`
- 다이얼로그: `role="alertdialog"`, `aria-modal="true"`, 열릴 때 포커스 트랩
- 탭: `role="tablist"` + `role="tab"` + `role="tabpanel"`
- 키보드: 탭 간 화살표 키 이동, 토글 Space 키, 라디오 화살표 키

## 9. API 연동

### 프로필 조회
```
GET /api/v1/users/me
Authorization: Bearer {accessToken}
```

### 프로필 수정
```
PATCH /api/v1/users/me
Authorization: Bearer {accessToken}
Body: { "nickname": "민지_취준생", "profileImageUrl": "..." }
```

### 북마크 목록 조회
```
GET /api/v1/bookmarks?page={page}&size=20
Authorization: Bearer {accessToken}
```

### 북마크 삭제
```
DELETE /api/v1/bookmarks/{bookmarkId}
Authorization: Bearer {accessToken}
```

### 알림 설정 조회
```
GET /api/v1/notifications/settings
Authorization: Bearer {accessToken}
Response: { "success": true, "data": { "emailEnabled": true, "daysBeforeDeadline": 7 } }
```

### 알림 설정 변경
```
PUT /api/v1/notifications/settings
Authorization: Bearer {accessToken}
Body: { "emailEnabled": true, "daysBeforeDeadline": 7 }
```

### 로그아웃
```
POST /api/auth/logout
Authorization: Bearer {accessToken}
```
